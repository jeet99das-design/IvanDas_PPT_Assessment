# Task 5 – Static Website Hosting using S3

# Objective
To host a static website using Amazon S3 without using EC2.

# Service Used
- Amazon S3
- Amazon Policy Generator

# Implementation Steps

1. Created S3 bucket.
2. Disabled block public access.
3. Uploaded website files (HTML, CSS, JS).
4. Enabled static website hosting.
5. Configured bucket policy for public read access through Amazon Policy Generator.
6. Accessed website using S3 endpoint URL.

# Key Learning

S3 can host static websites without the need for servers.  
Bucket policies are required to allow public access.
